# Himova — Supabase Setup

This folder holds the database schema and RLS policies for Himova.

## How to apply migrations (one-time setup)

1. Open the [Supabase SQL Editor](https://supabase.com/dashboard/project/bezdcjjkobhykjngegj/sql/new) for the Himova project.
2. Open `supabase/all.sql` from this repo.
3. Copy its full contents and paste them into the SQL Editor.
4. Click **Run**. You should see "Success. No rows returned" once it completes.

That's it — your database now has all tables, indexes, helper functions, RLS policies, and triggers.

## How to seed sample data (optional, for development)

After running `all.sql`, you can also paste the contents of `seed.sql` into the SQL Editor to insert one admin profile, three sample shopkeepers, and five sample products.

> **Note:** The seed script does NOT create auth users (those must be created via Supabase Auth). It only seeds catalog and shopkeeper rows.

## Migration files

The combined `all.sql` is generated by concatenating these in order:

1. `migrations/00001_init.sql` — extensions, helper functions, profiles
2. `migrations/00002_catalog.sql` — categories, products, variants, set types
3. `migrations/00003_shopkeepers.sql` — shopkeeper accounts and pricing
4. `migrations/00004_orders.sql` — orders and order items
5. `migrations/00005_stock.sql` — warehouse + shop stock + movement log
6. `migrations/00006_pos.sql` — POS sales, customers, custom products
7. `migrations/00007_engagement.sql` — notifications, rewards, app events
8. `migrations/00008_rls.sql` — Row Level Security policies
9. `migrations/00009_triggers.sql` — updated_at and profile-on-signup triggers

## Resetting the database

If you need to start over:

```sql
-- Run in SQL Editor. WARNING: drops all Himova data.
drop schema public cascade;
create schema public;
grant all on schema public to postgres, anon, authenticated, service_role;
```

Then paste `all.sql` again.
